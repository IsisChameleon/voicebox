B1self-muting tester

pipecat's default VAD turn-start strategy has enable_interruptions=True: the moment the bot's audio trips VAD, our in-flight Kokoro speech is cancelled and flushed — talk-over is physically impossible, and speak() already returned true.

B2serial command loop

bot.py awaits each command to completion before reading the next. A speak issued during a 30 s listen waits in the queue — the hard blocker for any interruption feature.

B3uncorrelated IPC

One shared cmd queue + one shared resp queue, no IDs. A cancelled MCP call leaves a stale response that the next command consumes — permanent off-by-one. Concurrent calls race for whichever response lands first.

B4errors as transcripts

Failures respond {"text": str(e)} — a Whisper hiccup reaches Claude as something the bot "said", then the loop breaks and the child is dead while the parent thinks it's alive.

B5fake-speech sentinel

On WS disconnect the child queues the literal string "I just disconnected, but I might come back." into the transcript queue — indistinguishable from real bot speech.

B6mic theft in the shim

Each getUserMedia(audio) creates a new track generator and reassigns the single micWriter. Apps that call it twice (mic check, reconnection) leave the in-call track silent, no error anywhere.

S0correctness fixes

Disable interruption-on-VAD (B1), survive command errors with proper error keys (B4), reap the child on stop + command deadline, shim multi-getUserMedia fan-out (B6), kill the fake-speech sentinel (B5, interim).smoke + e2e scripts pass; talk-over audio physically possible in the WAV.

S1full-duplex IPC

Correlation IDs on every request/response; child spawns a task per command; parent runs one response-router resolving per-id futures. Fixes B2 + B3 — speak during listen "just works".script: listen(30) + speak 2 s later → speech audible while listen still pending.

S2event-stream listen

Child keeps a timestamped log (bot_speech_started/stopped, transcript, tts_started/finished/interrupted, disconnects). listen(cursor) returns events-since; speak(wait=True) resolves at playout end. Highest-value single change: streaming-ish awareness + latency data for free.event timestamps match the merged WAV waveform.

S3declarative barge-in

speak(interrupt=True) and speak(when="bot_speaking", after_secs=1.5) — the child fires at audio-rate what Claude scripted in advance. Same pattern as Cekura's "Interrupter" persona. Yields the industry metric: interruption stoppage timing (~60 ms budget).overlap window visible in stereo recording; stoppage timing plausible vs waveform.

S4metrics & artifacts
